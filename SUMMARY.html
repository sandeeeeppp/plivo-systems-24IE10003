Caps: miss ≤ 1.00%, overhead ≤ 2.00×. Feedback lane = 0 B at this operating point; uplink is a flat 479 297 B every run. 81 ms is the measured floor; 90 ms is submitted — the floor padded ~10 ms for robustness against unseen profiles, same VALID result on A and B.

Why 81 ms is the floor

  • Each frame is sent as two independent datagrams at the same instant — independent relay draws, so the frame survives unless both drop, and the earliest copy carries delay min(d₁,d₂). Both land within one delay draw, so the delay floor equals the relay's delay_max, not delay_max + lag·20 ms.
  • Profile B's delay_max = 80 ms is the hard ceiling. At exactly 80 ms the ~0.7 ms pipeline overhead tips the near-max-delay frames past the deadline (13–15 miss); +1 ms clears them, leaving only the irreducible double-loss floor (~7 ≪ 15).
  • A retransmit costs a detection interval + two hostile lane crossings (~≥100 ms), so ARQ cannot beat either deadline — recovery is 100% redundancy-driven. ARQ is a backstop for looser hidden profiles.

The 2× budget forces a 1-byte sequence number

  • Cap 2.0× = 320 B/frame. Two 160-B payloads already spend it all — 100% redundancy leaves zero header room, so every header byte steals from redundancy.
  • Minimum copy = [seq:1][payload:160] = 161 B; the 1-byte seq is disambiguated against the live playout window (« 256 frames). Two copies = 322 B ⇒ governor skips the duplicate on only ~1.3% of frames.
  • The naïve 165-B chunk skips ~7% of duplicates → ~3% miss on A. Shrinking 165→161 B is what made A valid (0.07%).
  • A causal governor checks every send before transmitting, so up ≤ 1.998·160·n holds at every instant for any loss pattern — overhead ≤ 2.0× by construction.
